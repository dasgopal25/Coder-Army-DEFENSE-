Internet Explained from First Principles

1. Problem: Share Data Between Two Computers

Challenge: If you want two computers to exchange information, how do you connect them?

Solution: A direct physical link.

Example:

A simple connection can be made using a cable (like an Ethernet cable or fiber optic) between two computers. This allows them to send and receive signals directly without other devices in between.

Details:

  • The computers need compatible interfaces (Ethernet ports, USB ports, etc.).
  • Data flows along this link as electrical signals or light pulses.
  • This setup works for small-scale communication but isn't practical as the number of devices increases.

2. Problem: Connect Many Computers Efficiently

Challenge: What if you need more than two computers to communicate? How do you connect hundreds or thousands efficiently?

Solution: A local area network (LAN) with a central switch.

Example:

A home Wi-Fi router or an office network connects many devices.

Details:

  • A switch routes data between devices in the same network.
  • It ensures that data packets are sent only to the correct device, reducing unnecessary traffic.
  • It allows devices to share resources like printers and internet access.

3. Problem: Connect Many Networks Efficiently

Challenge: Networks exist everywhere—homes, businesses, universities. How do you connect all these separate networks together into one global network?

Solution: An internetwork connected by routers.

Example:

Your home network connects to the internet through your ISP’s router, which connects to other routers, forming a vast web.

Details:

  • Routers determine the best path for data packets.
  • They forward packets between networks until the destination is reached.
  • This layered approach allows scalability—you can connect millions or billions of devices without overwhelming the system.

4. Problem: Find a Specific Computer in This Massive Web

Challenge: With so many networks and devices, how do you know where to send data?

Solution: Every device gets a unique IP Address.

Example:

Like a postal address, an IP address tells the network where to send data.

Details:

  • IPv4 addresses look like this: 192.168.1.1.
  • IPv6 addresses, needed due to more devices, look like this: 2001:0db8:85a3:0000:0000:8a2e:0370:7334.
  • DNS (Domain Name System) helps translate human-readable names like www.example.com into IP addresses.

5. Problem: Send Data Reliably Without Errors

Challenge: Networks are noisy, unreliable, and can lose data. How do you ensure data arrives complete, in order, and without duplication?

Solution: A set of protocols, primarily TCP/IP, that handle data transmission.

Key Concepts:

  • Packetization: Data is broken into small chunks called packets, each with a header containing source and destination addresses.
  • Routing: Each packet may take a different path but is reassembled at the destination.
  • Error Checking: Checksums and acknowledgments ensure corrupted or missing packets are detected and retransmitted.
  • Flow Control: TCP adjusts the speed of data transmission based on network congestion.
  • Fairness: Protocols prevent one user from hogging all the bandwidth.

Final Thoughts

These fundamental problems and solutions form the backbone of how the internet operates:

  • Physical links connect computers.
  • Local networks and switches manage groups of devices.
  • Routers expand connectivity across the world.
  • IP addresses ensure data finds the right destination.
  • TCP/IP protocols guarantee that data arrives correctly, securely, and efficiently.

Additional Internet Concepts

1. Domain Name System (DNS)

How DNS Works:

  1. User enters domain → goes to Recursive Resolver.
  2. Recursive resolver queries Root Server.
  3. Root Server points to Top-Level Domain (TLD) Server (e.g., .com, .org).
  4. TLD Server points to Authoritative Server.
  5. Authoritative server provides the actual IP address of the domain.

2. MAC Address

MAC = Media Access Control Address

  • 48-bit permanent hardware address (unique to each device).

3. Ports

Port Number: 16-bit unsigned integer (0–65535).

Common ports:

  • HTTP (80): Not secure.
  • HTTPS (443): Secure.

4. IPv4

  • 32-bit address.
  • Range: 0.0.0.0 → 255.255.255.255.
  • Divided into 4 octets (8 bits each).
  • Total = 2³² ≈ 4.3 Billion addresses.

5. IPv6

  • 128-bit address.
  • Total = 2¹²⁸ addresses.
  • ≈ 340 undecillion addresses (virtually unlimited).

6. LAN (Local Area Network)

  • Connects multiple computers/devices in a limited area (home, office).
  • Diagram shows router connecting multiple devices.

7. VPN

VPN = Virtual Private Network

  • Provides secure, encrypted communication over the internet.

8. Web Development Basics

Client-Server Model

  • Client: sends request.
  • Server: sends response.

Technologies:

  • HTML: structure.
  • CSS: style.
  • JavaScript: functionality.
